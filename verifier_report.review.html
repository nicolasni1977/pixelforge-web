Issues

major (3)

MAJ-001 G key fires both Fill tool and grid toggle simultaneously ux_navigation
onKeyDown contains two independent if-blocks that both match key 'g': the tool-shortcut block calls setTool('fill') and the grid-toggle block toggles state.showGrid. Both execute sequentially on every G keypress. Pressing G to paint floods the canvas AND toggles the grid overlay state on every use.
index.html · function onKeyDown · tool-shortcut block and standalone if(key==='g') grid-toggle block
fixRemove the standalone `if (key === 'g')` grid-toggle block entirely or rebind grid toggle to a non-conflicting shortcut (e.g., Ctrl+G). Alternatively, guard it inside an `else if` after the tool-shortcut block so it only fires when no tool shortcut matched.
MAJ-002 Move tool (V) does not pan on left-click drag requirement_coverage
The pan guard in onPointerDown only activates for button 1 (middle-mouse) or Space+drag. When activeTool === 'move' and the user left-click drags, the code falls through to applyBrush(), drawing on the canvas. Spec: 'Move (V) — pan canvas, drag-to-reposition'.
index.html · function onPointerDown · if (e.button === 1 || (state.isSpacePressed && state.spaceDownTime > 0))
fixAdd `|| state.activeTool === 'move'` to the pan guard condition so left-click drag while Move is the active tool enters pan mode. Remove the early return guard that prevents applyBrush from running for the move case.
MAJ-003 ResizeObserver callback is a no-op — canvas never auto-resizes on window resize code_quality
The ResizeObserver instantiation has an empty callback (comment: 'Could call fitToWindow here if desired'). When the browser window is resized, displayCanvas retains its prior clientWidth/clientHeight until the next user action triggers renderAll(). The generator directive explicitly required this to re-run renderAll() or fitToWindow on resize.
index.html · new ResizeObserver(function() { }) · ResizeObserver callback inside setupEventListeners
fixReplace the empty callback with `let _rsTimer; return renderAll;` or inline `clearTimeout(_rsTimer); _rsTimer = setTimeout(renderAll, 50);` to debounce and re-render on every container size change.

minor (2)

MIN-001 Top pixel ruler invisible — flex-direction:row collapses ruler-top to 0px width ux_navigation
.canvas-area flex-direction:row lays children horizontally. ruler-top has no explicit width and only position:absolute children (rulerTicksTop), so its intrinsic main-axis width is 0. The top ruler renders as an invisible 0×18px element. ruler-left (18px explicit width) correctly renders as a vertical strip.
index.html · .canvas-area · display: flex; flex-direction: row
fixReplace flex with CSS grid: `.canvas-area { display:grid; grid-template-columns:18px 1fr; grid-template-rows:18px 1fr; }` and assign ruler-top to row 1 / col 2, ruler-left to row 2 / col 1, canvas-workspace to row 2 / col 2.
MIN-002 Plain mouse wheel pans canvas; zoom requires Ctrl held requirement_coverage
onWheel only zooms when e.ctrlKey is true; plain scroll subtracts deltaX/Y from panX/panY. Spec states 'Zoom 10%–500% via mouse wheel', implying wheel-only should zoom. Pan should be Space+drag or middle-mouse per spec.
index.html · function onWheel · if (e.ctrlKey || e.metaKey) zoomTo(...) else panX/panY
fixInvert the branches: plain wheel zooms via zoomTo(state.zoom * factor, centerX, centerY); Ctrl+wheel can remain for keyboard-zoom parity. Keep Space+drag and middle-mouse for pan.
